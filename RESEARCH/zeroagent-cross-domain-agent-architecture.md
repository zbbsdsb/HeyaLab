# ZeroAgent: Cross-Domain De-Roled Agent Architecture Design

## A. Automated Literature Search and Selection
- **Reed et al., 2022, "A Generalist Agent (Gato)"** -- Multimodal multi-task generalist policy network. Proposes the Gato model: a single network (single set of weights) can handle Atari, image captioning, dialogue, robotic arm manipulation, and other diverse tasks, determining output form based on context. **Core method**: multimodal input sequences are serialized into a unified Transformer structure, eliminating the need to design different networks for different tasks.
- **Gallouedec et al., 2024, "Jack of All Trades (JAT)"** -- Cross-domain Transformer reinforcement learning model. Proposes the JAT model, using a single Transformer to simultaneously handle sequential decision tasks, vision, and text tasks, achieving outstanding performance across multiple domains (CV/NLP/RL). **Core method**: generates a token embedding for each time step, extends attention range, and adds observation prediction auxiliary tasks; the model is smaller than Gato with significantly better cross-domain performance.
- **Fu et al., 2025, "CASH: Capability-Aware Shared Hypernetworks"** -- Capability-aware hypernetwork collaboration architecture. For heterogeneous multi-agent teams, proposes the CASH architecture: uses hypernetworks to map each agent's capability vector and observations to policy network weights, enabling dynamic behavior adaptation based on capabilities. **Core method**: employs shared encoders and hyper-adapters to dynamically generate decoder parameters from individual capabilities (such as speed, capacity, sensing radius) as input, achieving flexible and diverse collaborative behaviors with support for zero-shot generalization.
- **Tarasova et al., 2025, "Decentralized Adaptive Task Allocation"** -- Decentralized dynamic task allocation architecture. Proposes a two-layer adaptive controller architecture: lower-layer executors (heterogeneous LLMs) independently execute tasks, while an upper-layer controller predicts task parameters and selects candidate executors. **Core method**: the controller uses Simultaneous Perturbation optimization and consensus mechanisms, dynamically broadcasting tasks to the best-matching agent subset based on predictions of task characteristics (execution duration, deadlines) and real-time feedback, achieving efficient allocation without centralized coordination.
- **Jonnalagadda, 2026, "Modular Intelligence: Skill-Based Architecture"** -- Skill-centric monolithic generalist agent architecture. Critiques the poor scalability of traditional multi-agent systems, proposing to divide knowledge into modular "skills" and have a single frontier model (such as a large language model) learn to execute them. **Core method**: uses a "skill library" to manage domain knowledge, abstracting each solved problem into a new skill for capability expansion; the model itself serves as the "processor," skills as loadable applications, achieving cross-domain horizontal scaling and self-evolution.
- **Bergamaschi et al., 2025, "Fast, slow, and metacognitive thinking in AI (SOFAI)"** -- Metacognitive multi-agent architecture combining fast and slow thinking. Introduces "System 1" and "System 2" agents (corresponding to fast intuitive and slow reasoning types respectively) plus an independent metacognitive agent to decide when to invoke deep reasoning. **Core method**: multiple parallel agents: S1 quickly leverages experience for direct answers, S2 performs multi-step symbolic/search reasoning; the metacognitive agent decides whether to enable S2 based on candidate answer quality and resource availability, and online-updates the experience model. This structure supports emergent human-like behaviors (skill learning, cognitive control, adaptation).
- **Xu, 2025, "Agentic Metacognition"** -- Low-code agent metacognitive monitoring architecture. Addresses the problem of autonomous agents getting stuck or producing errors by proposing a secondary metacognitive layer that monitors the main execution process; upon detecting failure triggers such as high latency or repetitive actions, it proactively reports to humans and halts execution. **Core method**: the metacognitive agent performs real-time prediction and analysis of the main agent's "thinking" state; when it detects failure tendencies, it actively triggers human intervention and provides explanations, significantly improving overall reliability and user trust.
- **Hu et al., 2023, "Cross-Domain Zero-Shot Multi-Agent Collaboration (CDZSL)"** -- Cross-domain MAS method based on zero-shot learning. Addresses the scarcity of samples for new domain tasks by constructing a task description semantic space and using Feature Synthesis Mapping (FSM) to synthesize target domain features from source domains, guiding MAS decision-making in unknown environments. **Core method**: first decomposes source task states into stages to build a state transition graph, then constructs a task semantic embedding space to align source/target tasks; uses zero-shot learning to map seen-class features to unseen classes, achieving knowledge transfer and adaptive decision-making.

**Method Strengths and Weaknesses Analysis:**
- Cross-domain transfer capability: Generalist models like Gato/JAT demonstrate the possibility of shared weights and multimodal unification, but lack active selection and team collaboration; the Skill-Based approach emphasizes modular knowledge bases that can support arbitrary tasks, suggesting ZeroAgent could also build a capability library; zero-shot methods like CDZSL provide a path to sample-free transfer via semantic descriptions, usable for ZeroAgent's self-learning of new domains.
- Multi-agent collaboration strategy: CASH and Tarasova et al. focus on dynamic allocation and coordination in heterogeneous teams, highlighting the importance of capability matching; SOFAI's hierarchical structure separating control and execution provides a reference; although MetaAgent-FSM and similar auto-design approaches manually assign roles, they demonstrate the potential of free multi-agent combination. ZeroAgent will draw on the thinking of "dynamic teaming based on capabilities, autonomous collaboration."
- Adaptive/metacognitive mechanisms: SOFAI and Xu et al. introduce evaluation, monitoring, and learning modules that enable systems to reflect and optimize. ZeroAgent needs to combine these ideas: for example, using a metacognitive agent to evaluate current capability match, identify knowledge blind spots, and proactively invoke external tools or learning strategies. CASH uses hypernetworks to make policies adapt to capability changes; the Skill-Based architecture automatically expands the capability library through successful experience, also providing a path for self-evolution.

## C. ZeroAgent Architecture Design

- **Capability Vector Representation**: Maintain a multidimensional **capability vector** for each ZeroAgent, quantifying its knowledge domains, skill proficiency, innovation capacity, and adaptability (e.g., through neural network encoding or embedding representations). This can reference the continuous capability vectors used in CASH; each dimension can represent different knowledge domains or skill types.
- **Task Matching and Dynamic Scheduling**: Map **task requirements** to vectors as well (including domain, difficulty, resource needs, etc.). Use optimization algorithms (such as multi-objective programming) to compute the match between each agent's capability vector and task requirements, dynamically scheduling the best-matching agent for execution. Control-layer prediction methods can be adopted to estimate each task's parameters, with distributed optimization (such as SPSA consensus) used to decide allocation plans.
- **Collaboration and Team Formation Process**: When a complex task arrives, the ZeroAgent system automatically **forms a team**: first filtering candidate agent sets by match score, then these agents autonomously manage task allocation through communication protocols (such as broadcast or voting), similar to joint collaboration modes. Within the team, mixed collaboration topologies such as pipeline, hierarchical, or pooled can be adopted. For example, some agents handle solution planning (as "supervisors"), while others handle execution (as "workers"), as in a hierarchical structure; alternatively, multiple agents can freely discuss in parallel (brainstorming style). Process diagram:

```
[Task Arrival] --> [Capability Match Scheduling] --> [Temporary Team Formation] --> [Task Allocation & Execution]
         |                          ^                    |
         v                          |                    v
  [Feature Extraction/Objective Analysis]         [Collaboration Communication Protocol]         [Execution Result Recording]
         |                          |                    |
         v                          v                    v
   [Requirements vs Agent Capability Comparison] ------> [Multi-Agent Collaborative Execution] --> [Metacognitive Evaluation & Learning]
```

- **Metacognition and Self-Evolution Mechanism**: During execution, the system's metacognitive layer continuously **monitors performance** and self-evaluates: for example, comparing **expected vs actual** performance after each task, identifying **knowledge and capability gaps**. Referencing SOFAI's metacognitive architecture and Xu's approach, ZeroAgent can have a central or distributed metacognitive module that shares experience and updates models across all agents. When metacognition detects uncovered knowledge domains, it proactively initiates a **learning process**: invoking external tools or human knowledge sources (such as literature retrieval, APIs, expert systems, etc.) to fill gaps. Metacognition not only makes real-time decisions but also handles offline reflection. The overall flow: agent set <-> collaborative execution <-> metacognitive evaluation/feedback <-> capability vector update/expansion. Through iteration, this architecture surpasses traditional fixed-role systems in flexibility, cross-domain capability, and adaptability.

**Experiment design plan**:
- *Multi-task cross-domain testing*: Design task sequences across multiple different domains (e.g., image classification, logical reasoning, physics simulation, etc.). Compare ZeroAgent's performance against traditional specialized agents or fixed-role agents: focus on time to first successful task completion, success rate, and resource consumption. Test whether ZeroAgent can rapidly self-adapt and transfer knowledge on new domain tasks.
- *Collaboration efficiency metrics*: Measure team task completion efficiency (e.g., average completion time, success rate) and communication cost. Introduce team size and task complexity variations to evaluate the scalability of the dynamic teaming mechanism.
- *Self-optimization and metacognitive evaluation*: Observe the evolution of capability vectors and performance improvement over multiple rounds of tasks. Design quantifiable metrics (e.g., task success rate growth rate, error correction count) to verify the metacognitive layer's effectiveness in blind spot discovery and knowledge expansion. Knowledge tracing or blind spot testing (deliberately excluding certain domain tasks from the initial training set) can be used to verify ZeroAgent's ability to discover unknown domains.
- *Metric system*: Includes cross-domain transfer rate (performance on new domain tasks relative to baseline), team task completion rate, decision resource utilization (time/network calls, etc.), and self-improvement index (e.g., number of executions of the same task vs. success rate change). Ablation experiments (enabling only some components, e.g., disabling the metacognitive layer) are conducted using the control variable method to verify each mechanism's contribution.

## E. Innovation Points, Challenges, and Disruptive Value

- **Disruptive innovation**: ZeroAgent breaks the fixed-role paradigm, treating agents as "zero-role" entities with composable capability sets. Unlike traditional systems that predefine roles by function (e.g., "Chief Architect," "Assistant"), ZeroAgent autonomously undertakes tasks through capability vectors and goal-driven scheduling, eliminating dependence on preset roles/identities. It modularizes and libraries capabilities (drawing from the skill architecture), and achieves collaboration through dynamic allocation at the multi-agent level.
- **Cross-domain and adaptive leadership**: Through unified vector representation and metacognitive learning, ZeroAgent freely transfers knowledge across different domains without requiring specialized training for new domains. The metacognitive layer evaluates in real-time and invokes external knowledge sources, giving it human-like introspection and improvement capabilities. Multi-agent collaboration combines the advantages of decentralization and hierarchy, dynamically forming temporary teams to efficiently solve complex tasks (addressing the maintenance challenges caused by traditional division-of-labor refinement).
- **Challenges and solutions**: Implementing ZeroAgent faces difficulties including **high-dimensional capability space management**, **complex optimization scheduling**, and **real-time learning efficiency**. Dimensionality reduction and feature learning techniques can be used to manage capability vectors; approximate algorithms or reinforcement learning can be introduced for online optimization in task scheduling; the metacognitive layer can use incremental learning and memory modules (such as meta-reinforcement learning) to improve efficiency. Additionally, multi-agent system security and privacy must be ensured, drawing on cross-domain security analysis frameworks.
- **System value**: ZeroAgent has breakthrough advantages in cross-domain intelligence, role freedom, multi-agent collaboration, and self-evolution. Its capability-library architecture and collaboration mechanisms give the system enterprise-level scalability and interpretability; the metacognitive module improves reliability and continuous learning capability. Compared to current agent systems where each has its own role, ZeroAgent embodies true global optimization and creative solution-seeking, with the potential to become a next-generation general-purpose intelligence platform.

**Open questions and limitations**: ZeroAgent still needs to address research questions such as capability vector dimension design, task-capability mapping efficiency, and metacognitive decision thresholds. Subsequent work requires actual prototype validation of the above design and iterative improvement of mechanisms.
